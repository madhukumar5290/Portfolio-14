Accessibility

WCAG 2.1 AA in plain English

A practical eLearning-focused guide built from the four POUR principles.

Principle 1

Perceivable

Learners can perceive the content — through sight, hearing, or touch.

👁️

Colour contrast

Body text: 4.5:1 minimum. Large text (18pt+): 3:1. UI components: 3:1.

🖼️

Alt text

Decorative images: alt="". Informative: short description. Complex: long description.

💬

Captions

Every video. Open or closed. SRT, VTT, or burned-in for Storyline.

🎧

Audio descriptions

Narrate visual-only information for blind learners. Separate audio track or scripted pauses.

Principle 2

Operable

Learners can navigate and interact — including keyboard-only users.

⌨️

Keyboard access

Every interaction (button, dropdown, slider) reachable with Tab + Enter / Space.

🎯

Visible focus

Don't hide focus rings. Make them at least 2px and contrast 3:1.

⏰

No time traps

Avoid timed quizzes when possible. If unavoidable, allow extension.

🚫

No flashing

Nothing flashes more than 3 times per second. Photosensitive epilepsy risk.

Principle 3

Understandable

Content reads at the right level, behaves predictably, and helps learners recover.

🌐

Lang attribute

Set lang="en" on <html>. Mark mid-content language switches with lang="ar" etc.

🔁

Predictable

Navigation and patterns consistent across all screens. No surprise modals on focus.

🛟

Error help

Forms identify errors clearly, suggest fixes, and don't lose entered data.

📖

Plain language

Aim for grade 7–9. Use the AI Analyzer.

Principle 4

Robust

Works reliably with assistive technologies — screen readers, switch devices, voice control.

🔖

Semantic HTML

Use h1–h6, lists, buttons (not divs). Storyline uses ARIA labels — set them carefully.

📢

ARIA live regions

Announce dynamic content changes to screen-reader users without yanking focus.

🧪

Test with real AT

NVDA + Firefox, JAWS + Chrome, VoiceOver + Safari. At least two combinations.

Quick wins

10-minute accessibility audit

  1. Disable your mouse for 1 minute — try to complete a slide
  2. Run the WAVE browser extension on your published course
  3. Check colour contrast with the WebAIM Contrast Checker
  4. Turn on Windows High Contrast mode — does the course survive?
  5. Run the course at 200% zoom — anything cut off or overlapping?
  6. Read 3 random slides at grade level — Hemingway editor
  7. Test with NVDA + Firefox on the first 3 slides
  8. Watch a captioned video on mute — does the script make sense?
  9. Tab through the whole module — focus order logical?
  10. Ask one learner with a disability to pilot — and pay them
