ID Frameworks

Six models, one toolbox

When to use what — without dogma.

📐

ADDIE

The classic 5-phase model

Phases: Analyse → Design → Develop → Implement → Evaluate

Best for: well-scoped, compliance, regulated industries.

Pros: stakeholder-friendly, documented, predictable. Cons: linear, slow to iterate.

⚡

SAM (Successive Approximation Model)

Agile iterative cousin of ADDIE

Phases: Preparation → Iterative Design → Iterative Development

Best for: tight deadlines, evolving content, demos-first stakeholders.

Pros: fast prototypes, less rework. Cons: needs frequent stakeholder access.

🌱

Bloom's Taxonomy

Hierarchical model of cognitive skills

Phases: Remember → Understand → Apply → Analyse → Evaluate → Create

Best for: writing objectives that aren't "understand…"

Pros: universal vocabulary. Cons: easy to over-rely on action verbs alone.

9️⃣

Gagné's 9 Events of Instruction

A script for any lesson

Phases: Gain attention → Inform objective → Recall prior → Present → Guide → Practice → Feedback → Assess → Retention

Best for: ensuring no instructional step is skipped.

Pros: complete checklist. Cons: easy to apply mechanically.

📊

Kirkpatrick 4 Levels

Training evaluation model

Phases: Reaction → Learning → Behavior → Results

Best for: justifying training spend with leadership.

Pros: business-friendly. Cons: Levels 3 & 4 are expensive to measure.

🎯

Cathy Moore's Action Mapping

Performance-first design

Phases: Goal → Behaviors → Practice → Info

Best for: skills training, performance support, ditching "nice-to-know".

Pros: ruthless focus. Cons: less suitable for awareness-building.
